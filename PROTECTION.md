# PROTECTION — privacy & originality

Author: **Benden** · License: **CC BY 4.0** · Strategy: **public + timestamp + attribution**

This note records how the Genesis project is protected. The core principle:

> For an original conceptual framework, the strongest protection is **not hiding
> it — it is publishing it clearly, with attribution and a public timestamp.**
> Public commits prove *Benden proposed this, on this date*; CC BY makes
> attribution a *condition of use*. Concealment protects nothing here; clear
> public authorship protects everything that can be protected.

---

## 1. Originality & attribution (the real protection for a framework)

Legal reality to keep expectations correct:
- **Ideas are not protected by copyright; expression is.** The *concept* ("a world
  is generated by folding") can't be fenced off — but the **specific text, figures,
  and code** you wrote are protected, and **authorship/priority** can be established.
- So the goal is not "stop others using the idea" (impossible) but **"establish
  Benden as the first to publicly express it, and require attribution."**

What is in place / to keep in place:
- [x] **Public repo = strongest originality protection.** Every commit carries an
  immutable timestamp — public evidence that this framework was published by
  Benden on a given date. Publishing *beats* hiding for priority.
- [x] **CC BY 4.0** — others may use, adapt, even commercialize, but **must
  attribute Benden**. The smart posture for a concept: don't fight an unwinnable
  "ideas aren't protected" battle — make attribution a *condition*.
- [ ] **Embed attribution in every core file**, not just LICENSE: put
  `Author: Benden · CC BY 4.0 · first published 2026-06` at the top of README,
  each doc, and in code headers — so attribution travels with a file if copied out.
- [ ] **Keep the "history of corrections" long-form** (`TASKS.md` deliverable #1):
  besides being good content, the step-by-step derivation is the **strongest
  originality evidence** — an imitator cannot reproduce the *process*. The process
  is the fingerprint of authorship.
- [ ] *(optional, heavier)* a formal hash/timestamp service for extra evidence —
  GitHub public commits are usually enough at this stage; this is icing.

## 2. Privacy (conversation data)

The conceptual conversations that produced this framework are, by default on
consumer Claude plans, the kind of data that *can* be used for training unless
opted out. Actions:
- [ ] **Turn off the training toggle now.** Settings → Privacy Settings → turn off
  "help improve Claude" → save. Drops retention 5 years → 30 days and blocks the
  main training pipeline. (Verify current state — many are opted in by a default toggle.)
- **Know the limits:** opt-out is not absolute — conversations flagged for safety
  review may still be used regardless of the setting; deletion still has a ~30-day
  back-end tail. So **keep the core in your own controlled place (this repo),** not
  only in chat.
- [ ] For the most sensitive future ideation, consider **Incognito / temporary
  chats** (not written to account memory, generally not used for training).
- **The deepest privacy move is already done:** moving the canonical retention out
  of cloud chat and into a repo *you* control. Chats can be analyzed, retained, or
  lost on delete; **the repo you push is yours.**

---

## 3. The public/private tension (decided: PUBLIC)

- **Public** = strongest originality/attribution (the world sees the timestamp);
  anyone may read/use **with attribution to Benden**.
- **Private** = ideas don't circulate yet, *but commit timestamps still protect
  priority* — you can flip to public later.
- **Decision: PUBLIC.** Chosen to establish the Benden ↔ Genesis link openly and
  to get the strongest, public, tamper-evident priority. (If ever wanting to
  re-medidate before wider sharing, private→public later loses none of the
  timestamp protection.)
